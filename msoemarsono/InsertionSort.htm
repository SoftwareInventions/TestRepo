Insertion Sort

The insertion sort is a type of the many sorting algorithms. The insertion sort utilizes two different pointers (in this example, pointers are variables containing numbers which corresponds to an arrays node) simultaneously. One pointer marks the location of the number to be checked, called main number in this example. The second pointer marks the location of the number(s) main number will be checked against, named parameter.

The insertion sort would start with main number pointing to the second element of the array, and parameter pointing to the first element of the array. Next, we do a comparison of the two elements. If the element main number is pointing at is greater than the element parameter is pointing, we move on to the next iteration of this algorithm. If the element main number is pointing at is lesser than the element parameter is pointing, then a switch occurs.

A switchs goal is to swap the elements parameter and main number are pointing at. We can do this by declaring a temporary variable named tempVar. We store the element main number is pointing at in tempVar. If there are element(s) between parameter and main number, a relocation would happen, where these elements positions are shifted up by 1 position (the element main number is pointing at will be overwritten by the element which was previously in position main number - 1). After the necessary relocation, we replace the element parameter is pointing at to the value stored in tempVar. Next, we increment parameter by one and if parameter is pointing to the same array element as main number (which is the case in the first iteration), we end the iteration.

The next iteration of the insertion sort starts by incrementing main number by one (now main number is pointing to the third element of the array) and resetting parameter to point back to the first element of the array. Similar to the first iteration, we do a comparison between the element main number points at and the element parameter points and do the necessary switch and relocation. The entire sort will end when the value of main number is larger than the size of the array itself. At that point, the array is sorted.
