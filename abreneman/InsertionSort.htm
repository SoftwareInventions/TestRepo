Introduction to Insertion Sort

 

By Amanda Breneman

 

The Algorithm

 

Insertion sort is an algorithm that creates a sorted list by finding the correct place for each element one at a time. Much like you would sort a hand of cards, this algorithm looks at each element and finds its correct place in the list before moving onto the next element.

 

Imagine you are holding a hand of cards that you want to arrange in ascending numerical order. You start by looking at the second card from the left and comparing it to the first card. If the second card is smaller than the card to its left (the first card in the array), you swap them. Then it is time to look at the third card from the left: if the third card is smaller than the second card, the third card is swapped with the second card. This pattern continues until the cards are in order; this concept of sorting a hand of cards is very similar to the insertion sort algorithm.

 

The code example below is an implementation of insertion sort that will sort the elements of an array into ascending order. The code that implements this sort is composed of two loops: one loop to iterate through each element in the array and a second loop to move an element to its correct place, if it is not already there.

 

int a, b, temp;

for (a = 1; a < array.length; a++) {

temp = array[a];

b = a;

while (b > 0 && array[b-1] > temp) {

array[b] = array[b-1];

b--;

}

array[b] = temp;

}

 

The sort assumes the first element is already in sorted order, so the first loop starts with the second element of the array (which is at index 1), and sets temp = 2 in the example array shown below. Since the first element of the array is larger than the second – the 5 is larger than the 2 – the 2 gets shifted to the left and the 5 moves to the location that the 2 just vacated. At this point in the code, the second loop has been exited and it is time to begin the second iteration of the first loop. The first loop increments the location of the array element being sorted and sets temp = 4 since that is the next element to be sorted.

 

The next element, which is 4, needs to be moved one spot to the left to be in the correct location, between the 2 and 5. The code enters the second loop and moves the 4 one spot to the left. Since the element being sorted only needs to be moved one spot, the second loop exits after the first iteration and the 5 gets put in the third location, resulting in the first three elements being in sorted order: 2, 4, 5.

 

This pattern of evaluating an element and shifting it left until the element is in sorted order is followed until the last element in the list is placed in its correct location and the entire list is sorted.

The Complexity

 

Algorithms are rated on how quickly and efficiently they can be executed. The most commonly accepted method used to describe the efficiency or complexity of an algorithm is called Big O Notation. Big O notation describes the worst-case scenario, or the longest amount of time the algorithm could take to execute.

 

The complexity of the insertion sort algorithm is directly related to the amount of elements in the list being sorted, with the accepted complexity of O(n2). The one exception to this complexity is when the list is a best-case scenario.


Looking at a list that is made up of the elements [1, 2, 3, 4, 5], all it takes to sort this list is to look at each element and see that it is already in the correct location. This list is an example of a best-case scenario, since it is already sorted and the algorithm doesnt need to move any of the elements. The big O notation of this list is O(n), where n is the number of elements in the list. In this example, the list has 5 elements and the algorithm has to look at each of the 5 elements once before moving on.

 

When it comes to putting insertion sort into practice, the best-case scenario is not common and cannot be counted on to be the average efficiency of the insertion sort algorithm. Any situation other than the best-case scenario has a big O complexity of O(n2). Algorithms that use nested iterations (such as the two nested for loops used in the insertion sort) often have the complexity of O(n2).

 

Looking at a list that is made up of the elements [2, 3, 1, 4, 5], we see that it is not a best-case scenario since it is not already in sorted order. This list is an example of an average-case scenario. In this list, the 1 needs to be shifted left by two positions in order to numerically sort the list. When the algorithm gets to the 1 in the list, it determines that the 1 needs to be shifted left to be put in the correct location. At this point in the insertion sort code, the second for loop executes twice in order to move the 1 to the correct location. Since there are more steps required in this example than there are number of elements in the list, the complexity of this example is O(n2).

 

The graph below shows the curves created by the O(n) and O(n2) notations. The O(n) is a linear line that is directly proportional to the number of elements in a list. The O(n2) is a quadratic curve that is directly proportional to the square of the number of elements in a list.

 

Insertion sort is best used on smaller sets of numbers – O(n2) complexity is not very efficient when it comes to large sets of numbers.
