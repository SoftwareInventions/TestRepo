  • (Up to 250 words) Propose a one-page WorldFactory for GridWorlds, describing what your generated world will look like, with a brief description of the algorithm you'll use to generate it. This assignment is in the same spirit as the EC assignment at the end of 240, but with some important distinctions. First, it's an individual assignment, and must be distinctly different from any group project you did in 240. Second, it's about demonstrating some interesting code patterns, including loops, conditionals, etc, and writing very clean, well-documented, concise, properly styled, code. We'll be code-reviewing each submission, and ranking them by creativity of code, clarity and style, and conciseness. You need a design that's more than just some simple pattern-repetitions.

My generated world would be a graph of randomly distributed fields of grass, a new class of object called a ‘Barn’ and an ‘Astar’ cow. The Astar cow step method utilizes the heuristic of the A* algorithm to travel between fields of grass and eventually arrive at the ‘Barn’.  The A* algorithm implementation will include a priority queue data structure ( a queue ordered by the heuristics of distance and direction) and will navigate an array list based implementation of a graph data structure, using one array list for the edge set and one array list for the vertex set. The fields will be represented as the vertex set of the graph and their distances will be computed via the bottom left most grass object. The cow will move in a serpentine pattern through the grass, until it reaches the bottom left most grass object, at which point the step method will run the next iteration of the A* algorithm to determine which path to the next field has the lowest cost on the priority queue.
